javascript中根据变量的作用域可将变量分为全局变量和局部变量，在ES6中会新增块级作用域，以{}包含。

javascript引擎运行分为两步：预解析和代码执行。其中，预解析分为变量预解析(变量提升)和函数预解析(函数提升)。
(1) 变量提升就是把所有的变量声明提升到当前作用域的最前面，不提升赋值操作。
(2) 函数提升就是把通过function关键字声明的函数提升到当前作用域的最前面，不调用函数。
注意：不提升通过函数表达式声明的函数。

下面思考两个面试中常见的例题，帮助大家更好的理解javascript的预解析：
// 例1
var num = 10;
fun();
function fun() {
console.log(num);
var num = 20;
}
做这种输出结果的题目时，不能一头雾水地直接给个答案，正确的步骤是先模拟javascript引擎对代码进行预解析，拆开它的皮囊，以探其究竟。
所以我们不妨先写出预解析之后javascript引擎实际执行的代码，然后再给出结果，避免跳入题目的坑里。
// 例1实际执行的代码是：
var num; // 变量提升
function fun() { // 函数提升
var num; // 变量提升
console.log(num); // 从上往下执行，num变量只声明，没有赋值，输出undefined
num = 20;
}
num = 10;
fun();

// 例2
f1();
console.log(c);
console.log(b);
console.log(a);
function f1() {
var a = b = c = 9; // 此处通过连等实际只给变量a进行了声明，变量b和变量c只是赋值，函数内部的变量如果未声明，直接赋值，则会变成全局变量
console.log(a);
console.log(b);
console.log(c);
}
// 例2实际执行的代码：
function f1() {
var a = 9;
b = 9;
c = 9;
console.log(a); // 9
console.log(b); // 9
console.log(c); // 9
}
f1();
console.log(c); // 9
console.log(b); // 9
console.log(a); // a是函数内部的局部变量，外部输出会报错
